\subsection{Detailed Description}
Basic type definitions.

This header file defines the generic integer types that will be used for the implementation of hash functions; it also contains helper functions which encode and decode multi-\/byte integer values, using either little-\/endian or big-\/endian conventions.

This file contains a compile-\/time test on the size of a byte (the {\ttfamily unsigned char} C type). If bytes are not octets, i.\+e. if they do not have a size of exactly 8 bits, then compilation is aborted. Architectures where bytes are not octets are relatively rare, even in the embedded devices market. We forbid non-\/octet bytes because there is no clear convention on how octet streams are encoded on such systems.
